### アプリ利用者向けのクイズ機能の開発

#### 技術スタック

- Vue
  - Vue 2.5.17
  - @vue/cli v3
  - vuex 3.0.1
  - vue-router 3.0.1
- Firebase  
- Ruby on Rails

#### 詳細

- 企業コラボとしてクイズという機能を開発することになった
  - 諸般の事情で金銭が発生することはすぐにはなかったため、盛り上がれば今後も継続、かつマネタイズにつなげていくという感じ
- そのため、技術とプロダクト検証という方向性に合わせて、キャッチアップのハードルが低そうなVue + vuex + vue-routerとFirebaseを利用することに
- 自分がフロントエンド全般を担当
- アプリ上のWebViewで表示するSPAを開発
- @vue/cliを利用して雛形を作成し少しずつ改変
- Fluxアーキテクチャに少しだけ理解があり興味関心が強かったのでvuexを採用
- プロトタイプ的立ち位置なためNuxtとTS非採用(後に採用すべきだったと感じた)
- 初めてのVueだったが比較的スムーズに形にできて、後半にデザイナーやアプリチームと相談しながらブラッシュアップすることができた
- ゼロベースから作り上げていく/積み上げていくことが面白く感じた
- ESLint + Prettierを利用してコードベースをきれいにした
- [会社の技術ブログ](https://tech.studyplus.co.jp/entry/2019/01/28/094825)に投稿

### タイムゾーン撤廃

#### 技術スタック

- Ruby on Rails
- MySQL

#### 詳細

- 以前は国外もサポートしていたが、国内のみのサポートに切り替える
- そのため、タイムゾーンをユーザーごとに選択する機能が存在していたが、時関系のバグを引き起こす要因にもなっていたのでタイムゾーン選択を削除することに
- 日時に関する情報をインプットし、Railsで実装に落とし込む
- 日時をISO 8601に準拠した形でやり取りするようにクライアントチームと調整・修正
- 元々のタイムゾーン利用箇所を削除とテストの修正

### 塾・予備校向けのBtoBの開発

#### 技術スタック

- Ruby on Rails
- React
- Redux
- TypeScript
- Webpack

#### 詳細

- BtoCのプロダクトのチームからBtoBのプロダクトのチームに異動
- Webのみのサービスで、フロントエンドとバックエンドのどちらも触る
- React/Redux/TypeScriptを利用した静的型付け言語でのSPA開発を経験
- Ruby on Railsで使われていたwebpackerを引き剥がし、本当に必要な部分(カスタマーサクセスが利用する管理画面)のみにbootstrapだけ追加
- dependabotを利用した定期的なパッケージのバージョンアップ
